# Statistical Methods for Options Trading

**Prerequisite:** Probability theory, time series analysis, linear algebra
**Relevance:** Volatility forecasting, regime detection, dependency modeling, signal generation

---

## 1. Time Series Analysis

### Stationarity

A time series {X_t} is (weakly) stationary if:
- E[X_t] = mu (constant mean)
- Var(X_t) = sigma^2 (constant variance)
- Cov(X_t, X_{t+h}) = gamma(h) (autocovariance depends only on lag h)

Financial price series are non-stationary (trending). Log returns r_t = ln(P_t/P_{t-1}) are approximately stationary.

### Autocorrelation Function (ACF)

```
rho(h) = Cov(r_t, r_{t+h}) / Var(r_t)
```

For raw returns: ACF is approximately zero at all lags (weak-form efficiency). For squared returns (r_t^2) and absolute returns (|r_t|): Significant positive autocorrelation out to many lags (volatility clustering).

### ARMA Models

**AR(p) -- Autoregressive:**
```
X_t = c + phi_1*X_{t-1} + phi_2*X_{t-2} + ... + phi_p*X_{t-p} + epsilon_t
```

**MA(q) -- Moving Average:**
```
X_t = c + epsilon_t + theta_1*epsilon_{t-1} + ... + theta_q*epsilon_{t-q}
```

**ARMA(p,q) -- Combined:**
```
X_t = c + sum_{i=1}^p phi_i*X_{t-i} + epsilon_t + sum_{j=1}^q theta_j*epsilon_{t-j}
```

### ARIMA(p,d,q)

For non-stationary series, difference d times before fitting ARMA:

```
(1 - L)^d * X_t follows ARMA(p,q)
```

where L is the lag operator: L*X_t = X_{t-1}.

Box-Jenkins methodology:
1. **Identify:** Plot ACF, PACF; test for stationarity (ADF test); determine d
2. **Estimate:** Fit ARMA(p,q) to differenced series; use AIC/BIC for model selection
3. **Diagnose:** Check residuals are white noise (Ljung-Box test)
4. **Forecast:** Generate predictions with confidence intervals

### Application to Options

ARIMA is rarely used for price forecasting (returns are nearly unpredictable). Instead, it is useful for:
- Modeling implied volatility dynamics (IV series have serial correlation)
- Forecasting realized volatility
- Modeling spread dynamics (pairs trading)

## 2. GARCH Family Models

### GARCH(1,1)

The workhorse volatility model:

```
r_t = mu + epsilon_t,    epsilon_t = sigma_t * z_t,    z_t ~ N(0,1)

sigma_t^2 = omega + alpha * epsilon_{t-1}^2 + beta * sigma_{t-1}^2
```

Parameters:
- omega > 0: Baseline variance (intercept)
- alpha >= 0: Reaction to recent shocks (news coefficient)
- beta >= 0: Persistence of variance (memory coefficient)
- alpha + beta < 1: Stationarity condition

**Long-run variance:** V_L = omega / (1 - alpha - beta)

**Persistence:** alpha + beta (typically 0.95-0.99 for daily data)

**Half-life of vol shock:** ln(2) / ln(alpha + beta) trading days

### GJR-GARCH (Glosten-Jagannathan-Runkle, 1993)

Captures the leverage effect (negative returns increase vol more than positive returns):

```
sigma_t^2 = omega + (alpha + gamma * I_{epsilon<0}) * epsilon_{t-1}^2 + beta * sigma_{t-1}^2
```

where I_{epsilon<0} = 1 when epsilon_{t-1} < 0.

For equities, gamma is typically positive and significant, meaning downside volatility exceeds upside volatility.

### EGARCH (Nelson, 1991)

Models the log of variance (ensures positivity without parameter constraints):

```
ln(sigma_t^2) = omega + alpha * (|z_{t-1}| - E[|z|]) + gamma * z_{t-1} + beta * ln(sigma_{t-1}^2)
```

The gamma term captures asymmetry directly (positive gamma means negative returns increase vol).

### IGARCH (Integrated GARCH)

When alpha + beta = 1 (unit root in variance):

```
sigma_t^2 = omega + alpha * epsilon_{t-1}^2 + (1-alpha) * sigma_{t-1}^2
```

Special case: alpha = 1-lambda, omega = 0 gives EWMA (RiskMetrics). Vol shocks never decay -- the process has infinite memory.

### GARCH with Student-t Innovations

Replace z_t ~ N(0,1) with z_t ~ t_nu (Student-t with nu degrees of freedom):

```
f(z) = Gamma((nu+1)/2) / (sqrt(nu*pi)*Gamma(nu/2)) * (1 + z^2/nu)^{-(nu+1)/2}
```

Captures fat tails. Typical nu estimates for daily equity returns: 4-8 (significantly fatter tails than normal).

### HAR-RV (Heterogeneous Autoregressive Realized Volatility)

For high-frequency data:

```
RV_t^{(d)} = beta_0 + beta_d * RV_{t-1}^{(d)} + beta_w * RV_{t-1}^{(w)} + beta_m * RV_{t-1}^{(m)} + epsilon_t
```

where RV^{(d)}, RV^{(w)}, RV^{(m)} are daily, weekly, monthly realized volatility. Captures the multi-scale nature of vol dynamics. Often outperforms GARCH for vol forecasting.

### Model Comparison

| Model | Asymmetry | Fat Tails | Closed-Form | Typical Use |
|-------|-----------|-----------|-------------|-------------|
| GARCH(1,1) | No | With t-dist | Yes (approx) | Baseline vol forecasting |
| GJR-GARCH | Yes | With t-dist | Yes (approx) | Equity vol with leverage |
| EGARCH | Yes | Yes | No | Log-vol modeling |
| HAR-RV | No | Via residuals | Yes | High-frequency vol |

## 3. Regime Detection

### Hidden Markov Models (HMM)

**Model:** The observed returns are generated by a hidden (unobserved) state process:

```
State S_t in {1, 2, ..., K}        (hidden Markov chain)
r_t | S_t = k ~ N(mu_k, sigma_k^2)  (emission distribution)
```

**Transition matrix:** A_{ij} = P(S_t = j | S_{t-1} = i)

**Two-State Example (Bull/Bear):**

```
State 1 (Bull): mu_1 = +0.05%/day, sigma_1 = 0.8%/day
State 2 (Bear): mu_2 = -0.10%/day, sigma_2 = 2.0%/day

Transition matrix:
A = [0.98  0.02]   (Bull stays bull 98% of days)
    [0.05  0.95]   (Bear stays bear 95% of days)
```

**Estimation:** Baum-Welch algorithm (special case of EM):

1. **E-step (Forward-Backward):** Compute P(S_t = k | all observations) for each t
2. **M-step:** Update parameters {mu_k, sigma_k, A} given state probabilities
3. Iterate until convergence

**Inference:**
- **Filtering:** P(S_t = k | r_1, ..., r_t) -- real-time state estimation (forward algorithm)
- **Smoothing:** P(S_t = k | r_1, ..., r_T) -- retrospective state estimation (forward-backward)
- **Decoding:** Most likely state sequence (Viterbi algorithm)

### Markov-Switching GARCH

Combine regime switching with GARCH dynamics:

```
r_t = mu_{S_t} + sigma_t * z_t
sigma_t^2 = omega_{S_t} + alpha_{S_t} * epsilon_{t-1}^2 + beta_{S_t} * sigma_{t-1}^2
```

Each regime has its own GARCH parameters. The model captures both regime-dependent and within-regime volatility dynamics.

### Application to Options Trading

- **Regime-dependent strategy selection:** Use income strategies in low-vol regime, hedging strategies in high-vol regime
- **Regime-adjusted sizing:** Reduce position sizes when regime uncertainty is high
- **IV-HV spread analysis:** The relationship between implied and realized vol differs by regime
- **Tail risk management:** Increase hedging when the model assigns high probability to the crisis regime

## 4. Cointegration

### Concept

Two non-stationary time series X_t and Y_t are cointegrated if there exists a linear combination:

```
Z_t = Y_t - beta * X_t
```

that is stationary (I(0)). The series drift together in the long run.

### Engle-Granger Two-Step Method

1. **Regress:** Y_t = alpha + beta * X_t + epsilon_t (OLS)
2. **Test:** Apply ADF test to residuals epsilon_t. If residuals are stationary, the series are cointegrated.

Critical values are different from standard ADF (use Engle-Granger critical values).

### Johansen Method

For systems of more than two variables, test for multiple cointegrating relationships:

1. Estimate a Vector Error Correction Model (VECM)
2. Test the rank of the long-run multiplier matrix using trace and max-eigenvalue statistics
3. Rank r = number of cointegrating relationships (0 <= r <= n-1)

### Application: Pairs Trading with Options

1. Identify cointegrated pairs (e.g., GLD and GDX)
2. Compute the spread Z_t = Y_t - beta * X_t
3. When Z_t deviates from mean by > 2 sigma, expect mean reversion
4. Use options to express the view: buy call spread on the underperformer, buy put spread on the outperformer
5. Options provide defined risk and leverage vs. direct pair trading

## 5. Principal Component Analysis (PCA)

### Method

Decompose the covariance matrix of returns:

```
Sigma = V * Lambda * V^T
```

where Lambda = diag(lambda_1, ..., lambda_n) are eigenvalues (sorted descending) and V = [v_1, ..., v_n] are eigenvectors.

The i-th principal component is:

```
PC_i(t) = v_i^T * r_t
```

### Application to Implied Volatility Surface

Apply PCA to changes in implied volatility across strikes and expiries:

- **PC1 (~60-70%):** Parallel shift in the entire surface (level)
- **PC2 (~15-20%):** Skew change (OTM puts vs. OTM calls)
- **PC3 (~5-10%):** Curvature/smile change

### Application to Yield Curves

See fixed_income_options.md. PCA of yield changes gives level, slope, and curvature factors.

### Application to Portfolio Risk

Use PCA to identify the dominant risk factors in an options portfolio:
1. Compute the covariance of daily P&L contributions from each position
2. Extract PCs
3. If PC1 explains 80% of variance, the portfolio has one dominant risk (diversify)
4. Target portfolios where the leading PC explains < 50% (well-diversified)

## 6. Copulas for Dependency Modeling

### Motivation

Correlation (Pearson) measures linear dependence. Options payoffs are nonlinear, so we need a richer dependency structure. Copulas separate the marginal distributions from the dependency structure.

### Sklar's Theorem

For any joint distribution F(x, y) with marginals F_X(x) and F_Y(y), there exists a copula C such that:

```
F(x, y) = C(F_X(x), F_Y(y))
```

### Common Copulas

**Gaussian Copula:**
```
C(u, v) = Phi_2(Phi^{-1}(u), Phi^{-1}(v); rho)
```

where Phi_2 is the bivariate normal CDF and rho is the correlation parameter. No tail dependence (underestimates joint extreme events).

**Student-t Copula:**
```
C(u, v) = t_{2,nu}(t_nu^{-1}(u), t_nu^{-1}(v); rho)
```

Has symmetric tail dependence. Better for modeling joint crashes. Tail dependence coefficient: lambda = 2*t_{nu+1}(-sqrt((nu+1)(1-rho)/(1+rho))).

**Clayton Copula:**
```
C(u, v) = (u^{-theta} + v^{-theta} - 1)^{-1/theta}
```

Lower tail dependence (joint crashes) but no upper tail dependence. Good for modeling correlated drawdowns.

**Gumbel Copula:**
```
C(u, v) = exp(-((-ln u)^theta + (-ln v)^theta)^{1/theta})
```

Upper tail dependence (joint rallies) but no lower tail dependence.

### Application to Options

- Model the joint distribution of two underlyings for pricing basket options, worst-of options
- Estimate correlation risk in dispersion trades
- Stress testing: use Clayton copula to model increased dependence in crashes
- The Gaussian copula's failure to capture tail dependence was a contributing factor in the 2008 financial crisis (CDO mispricing)

## 7. Extreme Value Theory (EVT)

### Motivation

Normal and even Student-t distributions underestimate the probability of extreme events. EVT provides a rigorous framework for modeling the tails of distributions.

### Generalized Extreme Value Distribution (GEV)

The distribution of the maximum of n iid random variables converges to:

```
G(x) = exp(-(1 + xi*(x-mu)/sigma)^{-1/xi})
```

where xi is the shape parameter:
- xi > 0: Frechet (heavy tail, polynomial decay) -- financial returns
- xi = 0: Gumbel (light tail, exponential decay) -- normal-like
- xi < 0: Weibull (bounded tail)

### Generalized Pareto Distribution (GPD)

For exceedances over a high threshold u:

```
P(X - u <= x | X > u) -> H(x) = 1 - (1 + xi*x/sigma_u)^{-1/xi}
```

### Peaks Over Threshold (POT) Method

1. Choose a high threshold u (e.g., 95th percentile of losses)
2. Fit GPD to exceedances over u
3. Use the fitted GPD to estimate extreme quantiles (VaR, ES)

### Application

- Estimate the probability of 10-sigma events (which occur far more often than Gaussian predicts)
- Compute extreme VaR/ES for tail risk management
- Calibrate tail risk hedging strategies (how much OTM to go for tail hedges)

## 8. Jump-Diffusion Models

### Merton Jump-Diffusion (1976)

```
dS/S = (mu - lambda*k) dt + sigma dW + J dN
```

where:
- N is a Poisson process with intensity lambda (average number of jumps per year)
- J is the jump size: ln(1+J) ~ N(mu_J, sigma_J^2)
- k = E[J] = exp(mu_J + sigma_J^2/2) - 1

### Option Pricing Under Merton

Option price is a weighted sum of Black-Scholes prices:

```
C_Merton = sum_{n=0}^{infinity} (exp(-lambda'*T) * (lambda'*T)^n / n!) * C_BS(sigma_n, r_n)
```

where:
- lambda' = lambda * (1 + k)
- sigma_n^2 = sigma^2 + n*sigma_J^2/T
- r_n = r - lambda*k + n*ln(1+k)/T

This produces a leptokurtic implied vol smile (higher vol for deep OTM options).

### Kou Double-Exponential Jump-Diffusion (2002)

Jump sizes follow a double-exponential (asymmetric Laplace) distribution:

```
f_J(x) = p * eta_1 * exp(-eta_1*x) * 1_{x>0} + (1-p) * eta_2 * exp(eta_2*x) * 1_{x<0}
```

Parameters:
- p: Probability of upward jump
- eta_1: Rate of exponential decay for upward jumps (1/eta_1 = mean up-jump size)
- eta_2: Rate of exponential decay for downward jumps (1/eta_2 = mean down-jump size)

Advantage over Merton: memoryless property of exponential distribution allows closed-form solutions for many exotic options (barriers, lookbacks).

### Calibration

Fit jump parameters to the implied vol smile:
- lambda and sigma_J control the overall curvature of the smile
- mu_J controls the skew (asymmetry)
- The diffusion sigma controls the ATM level

## 9. Integration with Trading System

### Statistical Signal Pipeline

```
Raw Data -> Feature Engineering:
  1. Compute realized vol (Yang-Zhang, Garman-Klass)
  2. Fit GARCH model -> forecast vol
  3. Run regime detection (HMM) -> current regime probability
  4. Check cointegration relationships -> mean-reversion signals
  5. Compute IV surface PCA -> detect surface dislocations
  6. Estimate tail risk (EVT) -> adjust hedges

-> Signal Generation:
  - Vol trade signal: GARCH forecast vs. IV (rich/cheap)
  - Regime signal: Strategy selection based on regime
  - Spread signal: Cointegration-based pair trades
  - Tail signal: EVT-based hedge adjustment
```

### Model Selection Criteria

| Criterion | Formula | Use |
|-----------|---------|-----|
| AIC | -2*ln(L) + 2*k | Balance fit and parsimony |
| BIC | -2*ln(L) + k*ln(n) | Stronger penalty for complexity |
| Cross-validation | Out-of-sample MSE | Predictive performance |
| Diebold-Mariano | Compare forecast errors | Hypothesis test for forecast comparison |
