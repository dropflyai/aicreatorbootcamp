# Events — Authoritative Module

Events are the heartbeat of community. They create shared experiences,
forge relationships that asynchronous interaction cannot replicate, and
generate energy that sustains the community between gatherings. This
document codifies the design, execution, and measurement of community
events across all formats.

## 2. VIRTUAL MEETUPS

### Virtual Meetup Design

The most common community event format. Done well, virtual meetups create
consistent engagement. Done poorly, they become unwatched webinars.

**Structure (60-Minute Meetup):**
```
0:00–0:05  Welcome and housekeeping
0:05–0:10  Community updates / announcements
0:10–0:30  Featured content (talk, demo, interview)
0:30–0:45  Interactive segment (Q&A, discussion, exercise)
0:45–0:55  Open networking / breakout rooms
0:55–1:00  Wrap-up, next event preview
```

**Virtual Meetup Best Practices:**
1. **Start on time, end on time.** Respect is the foundation of repeat attendance.
2. **Cameras encouraged, not required.** Reduce barriers to participation.
3. **Interactive every 10 minutes.** Polls, chat prompts, reactions, breakouts.
4. **Feature members, not staff.** Member-presented content drives ownership.
5. **Record and share.** Async access respects timezone diversity.
6. **Consistent schedule.** Same day, same time, every occurrence.

### Virtual Meetup Metrics

| Metric | Definition | Target |
|--------|-----------|--------|
| Registration rate | Registrations / invitations sent | >10% |
| Attendance rate | Attendees / registrations | >50% |
| Engagement rate | Active participants (chat, voice) / attendees | >40% |
| Return rate | % attending 2+ events in series | >30% |
| NPS | Post-event satisfaction | >50 |

## 3. HACKATHONS

### Hackathon Design Framework

Hackathons are high-energy events where participants build projects in
a compressed timeframe. They produce innovation, deepen skills, and
create powerful shared memories.

**Planning Timeline:**
| Phase | Timing | Activities |
|-------|--------|-----------|
| Planning | 8–12 weeks before | Theme, rules, prizes, platform, sponsors |
| Promotion | 6–8 weeks before | Announce, recruit, create hype |
| Registration | 4–6 weeks before | Sign-up, team formation, idea submission |
| Preparation | 1–2 weeks before | Kickoff info, resources, mentors assigned |
| Execution | Event days | Hacking, mentoring, check-ins, demos |
| Judging | Last hours | Presentations, judging, awards |
| Follow-up | 1–2 weeks after | Showcase, blog post, project continuation |

**Hackathon Structure (48-hour example):**
```
Day 0 (Evening):
├── Kickoff ceremony (theme reveal, rules, inspiration)
├── Team formation (for individuals seeking teams)
└── Hacking begins

Day 1:
├── Morning check-in (progress updates)
├── Mentor office hours (experts available)
├── Mid-day workshop (relevant skill session)
├── Evening social (virtual or in-person)
└── Hacking continues

Day 2:
├── Morning: Final hacking push
├── Afternoon: Project submission deadline
├── Demo presentations (3–5 min each)
├── Judging deliberation
└── Awards ceremony and celebration
```

**Judging Criteria (Weighted):**
| Criterion | Weight | Description |
|-----------|--------|-------------|
| Innovation | 25% | Novelty of the idea and approach |
| Execution | 25% | Quality of implementation |
| Impact | 20% | Potential value to users/community |
| Presentation | 15% | Clarity of demo and explanation |
| Creativity | 15% | Surprise, delight, originality |

**Prize Strategy:**
- Grand prize should be meaningful but not so large it becomes the primary motivation
- Category prizes (best design, most innovative, best newcomer) broaden recognition
- Everyone-gets-something (participation certificate, swag) encourages future participation
- Non-monetary prizes (mentorship, featured project, conference ticket) often more valued

## 4. UNCONFERENCES

### Unconference Design

Unconferences are participant-driven events where attendees set the agenda.
Based on Open Space Technology (Harrison Owen, 1985).

**Core Principles:**
1. **Whoever comes is the right people.** Quality over quantity.
2. **Whatever happens is the only thing that could have.** Release control.
3. **Whenever it starts is the right time.** Organic flow.
4. **When it is over, it is over.** Natural endings.
5. **The Law of Two Feet:** If you are not learning or contributing, move.

**Unconference Structure:**
```
Opening Circle (30 min):
├── Welcome and principles explanation
├── Marketplace: Participants propose sessions (2-min pitches)
└── Session board filled (time slots x rooms)

Session Blocks (3–4 blocks, 45–60 min each):
├── Participant-facilitated discussions
├── Multiple parallel sessions
├── Notes captured in shared documents
└── Participants move freely between sessions

Closing Circle (30 min):
├── Session highlights shared (1 min each)
├── Action items and commitments
└── Feedback and next steps
```

### Virtual Unconference Adaptations

- Use breakout rooms as "rooms" in the session board
- Digital marketplace (Miro board, Google Sheet) for session proposals
- Shorter sessions (30–40 min) for virtual attention spans
- Dedicated facilitator to manage room transitions
- Shared note-taking document per session

## 5. ANNUAL SUMMITS

### Summit Design

The flagship community event. Annual summits are the highest-investment,
highest-impact events in the community calendar.

**Summit Components:**
| Component | Duration | Purpose |
|-----------|----------|---------|
| Keynote | 30–60 min | Inspiration, vision, state of community |
| Talks | 20–30 min each | Knowledge sharing, storytelling |
| Workshops | 60–120 min | Hands-on skill building |
| Panels | 45–60 min | Multiple perspectives on a topic |
| Networking | Dedicated blocks | Relationship building |
| Unconference | Half-day track | Participant-driven discussion |
| Awards/celebration | 30–60 min | Recognition, community identity |
| Social events | Evening | Fun, informal bonding |

**Summit Planning Timeline:**
| Months Before | Activities |
|---------------|-----------|
| 12 | Set date, secure venue (if in-person), budget |
| 9 | Open CFP (call for proposals), sponsorship outreach |
| 6 | Speaker selection, program finalization, registration open |
| 4 | Logistics finalization, content review, promotion push |
| 2 | Final speaker preparation, attendee communications |
| 1 | Rehearsals, final logistics, emergency planning |
| Event | Execute with military precision and warm hospitality |
| +1 week | Thank-you messages, feedback survey, content publication |
| +1 month | Impact report, sponsor report, retrospective |

### Summit Budget Framework

**In-Person Summit (200 attendees):**
| Item | Cost Range | Notes |
|------|-----------|-------|
| Venue | $10K–$50K | Location-dependent |
| Catering | $50–$150/person | $10K–$30K total |
| A/V and production | $5K–$25K | Sound, screens, recording |
| Speaker travel | $10K–$30K | If covering travel |
| Swag and materials | $5K–$15K | Badges, programs, gifts |
| Marketing | $2K–$10K | Promotion, design |
| Staff travel | $5K–$15K | If team travels |
| Contingency | 15% of total | Always budget this |
| **Total** | **$50K–$200K** | |

**Virtual Summit (500+ attendees):**
| Item | Cost Range | Notes |
|------|-----------|-------|
| Platform (Hopin, etc.) | $2K–$10K | Depends on features |
| Production | $5K–$15K | Graphics, streaming |
| Speaker fees | $0–$10K | Most community speakers donate |
| Marketing | $1K–$5K | Promotion, design |
| Swag (shipped) | $10K–$20K | If sending physical items |
| **Total** | **$20K–$60K** | |

## 6. HYBRID EVENTS

### Hybrid Event Design Principles

Hybrid events serve both in-person and virtual audiences simultaneously.
They are the hardest format to execute well.

**Core Challenge:** Virtual attendees must not feel like second-class citizens.

**Hybrid Design Rules:**
1. **Design for virtual first, add in-person.** Not the reverse.
2. **Dedicated virtual facilitator.** Not the same person running in-person.
3. **Equalized interaction.** Virtual participants can ask questions equally.
4. **Camera always on stage.** Virtual must see what in-person sees.
5. **Chat integration.** In-person screens show virtual chat.
6. **Separate networking.** Virtual breakouts + in-person networking.

### Technology Requirements

| Component | Purpose | Examples |
|-----------|---------|---------|
| Streaming platform | Broadcast to virtual | StreamYard, OBS, Zoom |
| Event platform | Virtual experience | Hopin, Run The World |
| Production | Multi-camera, graphics | Professional AV team |
| Audio | Clear in-room + stream | Wireless mics, audio mixer |
| Chat | Real-time interaction | Platform chat, Discord |
| Q&A | Question management | Slido, platform Q&A |

## 7. COMMUNITY-LED EVENTS

### Enabling Member-Organized Events

The most scalable event strategy is empowering members to organize their
own events under the community umbrella.

**Community-Led Event Framework:**

| Element | Community Provides | Member Provides |
|---------|-------------------|----------------|
| Brand | Logo, templates, guidelines | Local adaptation |
| Platform | Streaming/meeting account | Content and facilitation |
| Promotion | Cross-promotion in community | Local promotion |
| Support | Event planning guide, mentorship | Execution |
| Budget | Micro-grants ($100–$500) | Volunteer time |

**Quality Gates for Community-Led Events:**
- Topic aligns with community purpose
- Organizer is an active, trusted member
- Code of conduct will be enforced
- Event is registered in community calendar
- Post-event report submitted

### Regional Chapters

For geographically distributed communities, regional chapters create
local connection:

**Chapter Structure:**
- Chapter lead (appointed or elected)
- Minimum 10 active members per chapter
- Monthly local meetup (virtual or in-person)
- Quarterly coordination with global community
- Annual chapter leader summit

### Event Promotion Timeline

| Timing | Action | Channel |
|--------|--------|---------|
| 4 weeks before | Announcement | Community, email, social |
| 3 weeks before | Speaker/topic details | Community, social |
| 2 weeks before | Registration push | Email, community |
| 1 week before | Final reminder | Email, community, social |
| 1 day before | Last chance + logistics | Email, community |
| Day of | "Happening now" | Community, social |
| Day after | Recording + recap | Community, email |

### Post-Event Follow-Up

Within 48 hours of every event:
1. Send thank-you to attendees with recording link
2. Share key takeaways in community
3. Publish recording (edited if needed)
4. Send feedback survey
5. Analyze metrics
6. Retrospective with organizing team

## 10. ANTI-PATTERNS

| Anti-Pattern | Description | Fix |
|-------------|-------------|-----|
| Webinar masquerade | One-way presentation called "event" | Interactive design |
| Over-programming | Too many events, community fatigue | Quality over quantity |
| Speaker-only value | Only speakers benefit from events | Design for audience outcomes |
| Timezone tyranny | Same timezone for every event | Rotate or provide async |
| No follow-up | Event ends, nothing happens | Post-event workflow |
| Tech over connection | Focus on platform, not people | Human-first design |

---

**Events are the community's campfire. They are where stories are told,
relationships are forged, and identity is reinforced. Every event should
leave members energized, connected, and eager for the next gathering.
If an event does not do this, redesign it or cancel it.**
