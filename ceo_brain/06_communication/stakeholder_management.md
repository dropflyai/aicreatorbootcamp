# Stakeholder Management -- Board, Investors, Employees, and Beyond

## Overview

The CEO is the nexus of all organizational communication. Every stakeholder
group -- board, investors, employees, customers, partners, press, and
public -- receives its understanding of the company primarily through the
CEO's words and actions. Effective stakeholder management is not about
spin or impression management; it is about maintaining trust through
transparency, consistency, and authenticity with each audience.

This module codifies the frameworks for managing each stakeholder
relationship, the communication cadence for each, and the common failure
modes that destroy stakeholder trust.

---

## Board Management

### The Board Relationship

The CEO-board relationship is the most consequential professional
relationship in the company. Get it right: the board provides strategic
counsel, network access, and governance. Get it wrong: the board becomes
an adversary, a micromanager, or (worst case) fires the CEO.

### Board Communication Cadence

```
QUARTERLY:
+--------------------------------------------------+
| Board meeting (3-4 hours)                        |
|   - Financial review                             |
|   - Business update                              |
|   - Strategic discussion (not just reporting)    |
|   - Closed session                               |
+--------------------------------------------------+

MONTHLY:
+--------------------------------------------------+
| Board update (written, 1-2 pages)                |
|   - Key metrics with trend                       |
|   - Wins and challenges                          |
|   - Decisions made and rationale                 |
|   - Help needed from board                       |
+--------------------------------------------------+

AS NEEDED:
+--------------------------------------------------+
| Individual board member calls                    |
|   - Specific expertise requests                  |
|   - Pre-meeting alignment on sensitive topics    |
|   - Crisis notification                          |
+--------------------------------------------------+
```

### Board Management Principles

1. **No surprises** -- the board should never learn important news at
   the board meeting. Major developments are communicated individually
   beforehand.
2. **Bad news travels fast** -- deliver bad news immediately and with a
   plan. The CEO who hides bad news loses the board's trust permanently.
3. **Ask for help** -- boards want to be useful. Come with specific asks:
   introductions, strategic input, expertise.
4. **Manage up** -- prepare the board. Share context before meetings.
   Frame strategic discussions with clear options.
5. **Do not manage by consensus** -- the board advises; the CEO decides.
   Asking the board to make operational decisions is abdicating CEO
   responsibility.

### The Board Presentation Framework

```
BOARD DECK STRUCTURE (10-15 slides)
+--------------------------------------------------+
| 1. Executive Summary (1 slide)                   |
|    - Overall health: Green/Yellow/Red            |
|    - Top 3 things going well                     |
|    - Top 3 challenges                            |
|                                                  |
| 2. Financial Review (2-3 slides)                 |
|    - Revenue vs. plan                            |
|    - Burn rate and runway                        |
|    - Key unit economics                          |
|                                                  |
| 3. Product and Customers (2-3 slides)            |
|    - Shipping velocity                           |
|    - Customer growth and retention               |
|    - NPS/satisfaction                            |
|                                                  |
| 4. Team (1-2 slides)                            |
|    - Key hires and departures                    |
|    - Team health and engagement                  |
|    - Organizational changes                      |
|                                                  |
| 5. Strategic Discussion (2-3 slides)             |
|    - One or two strategic topics                 |
|    - Clear options with tradeoffs                |
|    - Specific input requested                    |
|                                                  |
| 6. Asks (1 slide)                               |
|    - What the CEO needs from the board           |
+--------------------------------------------------+
```

## Investor Relations

### Pre-Funding Investors (Prospective)

```
INVESTOR COMMUNICATION LIFECYCLE:
1. WARMING: Share company updates, build relationship
2. PITCHING: Formal pitch, data room, due diligence
3. CLOSING: Term sheet negotiation, legal
4. POST-CLOSE: Ongoing reporting and relationship
```

### Post-Funding Investors (Existing)

Investor update frequency and content:

```
MONTHLY INVESTOR UPDATE (Email, 1-2 pages)
+--------------------------------------------------+
| SUBJECT: [Company] - [Month] Update             |
|                                                  |
| HIGHLIGHTS:                                      |
| - [Top win 1]                                   |
| - [Top win 2]                                   |
| - [Top win 3]                                   |
|                                                  |
| LOWLIGHTS:                                       |
| - [Challenge 1 and how we are addressing it]    |
| - [Challenge 2 and how we are addressing it]    |
|                                                  |
| KEY METRICS:                                     |
| - Revenue: $X (vs. $Y last month, $Z plan)     |
| - Users: X (vs. Y last month)                  |
| - Churn: X% (vs. Y% last month)               |
| - Burn: $X/mo, Runway: Y months               |
|                                                  |
| ASKS:                                            |
| - [Specific introduction or help needed]        |
| - [Specific expertise needed]                   |
|                                                  |
| TEAM:                                            |
| - [Notable hires or changes]                    |
+--------------------------------------------------+
```

### Investor Communication Principles

1. **Consistency** -- send updates on the same schedule without fail,
   even when news is bad (especially when news is bad)
2. **Honesty** -- investors can handle bad news; they cannot handle
   dishonesty
3. **Conciseness** -- investors read dozens of updates; make yours scannable
4. **Asks** -- always include asks; investors want to help
5. **Metrics first** -- lead with numbers, follow with narrative

## Employee Communication

### The Employee Communication Stack

```
DAILY:
  Slack/Teams: Real-time coordination and information sharing
  Dashboard: Metrics visible to all

WEEKLY:
  Team standups: Status, blockers, priorities
  CEO email (optional): Brief context-setting for the week

MONTHLY:
  All-hands meeting: State of the company, strategy, Q&A
  Written update: For those who cannot attend

QUARTERLY:
  OKR review: How did we do? What is next?
  Skip-level meetings: CEO hears from non-direct reports

ANNUALLY:
  Company kickoff: Strategy, vision, goals for the year
  Strategy offsite: Deep strategic alignment (leadership)
```

### The Transparency Spectrum

```
FULL TRANSPARENCY       SELECTIVE SHARING       CONFIDENTIAL
(Share with everyone)   (Share with leaders)    (CEO/Board only)
- Company metrics       - Individual comp data  - M&A discussions
- Strategy and OKRs     - PIP/termination plans - Board dynamics
- Revenue numbers       - Fundraising timing    - CEO personal issues
- Customer wins/losses  - Specific client names  - Legal threats
- Hiring plans          - Executive candidates   - Regulatory issues
- Product roadmap       - Sensitive partnerships - Layoff planning
```

### Communication During Change

When communicating significant changes (reorgs, pivots, layoffs):

```
CHANGE COMMUNICATION PROTOCOL
1. PREPARE: Draft communication, anticipate questions, prep leaders
2. LEADERS FIRST: Brief executive team and managers before all-hands
3. ALL-HANDS: CEO presents the change (see below for format)
4. MANAGERS: Each manager discusses with their team (same day)
5. Q&A: Open channel for questions (Slack, email, follow-up meeting)
6. FOLLOW-UP: Written summary sent to all (for reference)
7. CHECK-IN: CEO or managers check in 1 week later
```

Change communication format:
1. **Why** -- the context and reasoning (be honest about the business case)
2. **What** -- what specifically is changing
3. **Who** -- who is affected and how
4. **When** -- timeline for the change
5. **What is not changing** -- provide stability anchors
6. **How we will support** -- specific support for affected people
7. **Q&A** -- open, honest, no deflection

## Customer Executive Relationships

### The CEO's Customer Role

The CEO should maintain direct relationships with:
- Top 10 customers by revenue
- Top 5 customers by strategic importance
- Any customer at risk of churning (escalation path)
- Prospective customers in enterprise deals

### Customer Communication Framework

```
CADENCE:
- Top customers: Quarterly business review (QBR)
- Strategic customers: Monthly check-in
- At-risk customers: Weekly until stabilized
- Prospective enterprise: As needed during sales cycle

CONTENT:
- Product roadmap (selective sharing)
- Company direction and vision
- Their feedback and priorities
- Partnership opportunities
- Issue resolution (escalation path)
```

### The Executive Sponsor Program

For top accounts, assign executive sponsors:

```
EXECUTIVE SPONSOR RESPONSIBILITIES:
1. Quarterly face-to-face (or video) with customer exec
2. Understand customer's business priorities
3. Ensure internal alignment on customer needs
4. Escalation path for critical issues
5. Strategic relationship building (long-term)
```

## Press and Public Communication

### Media Strategy

```
MEDIA COMMUNICATION LEVELS:
+--------------------------------------------------+
| PROACTIVE: Company-initiated press (launches,    |
|   milestones, funding, thought leadership)       |
|   CEO involvement: High                          |
+--------------------------------------------------+
| REACTIVE: Responding to media inquiries          |
|   CEO involvement: Selective (major stories only)|
+--------------------------------------------------+
| CRISIS: Managing negative press                  |
|   CEO involvement: Always (see crisis_mgmt.md)   |
+--------------------------------------------------+
```

### Media Interaction Principles

1. **Know your message** -- prepare 3 key points before any interaction
2. **Stay on message** -- do not be drawn into topics you do not want
   to discuss
3. **Nothing is off the record** -- assume everything you say will be
   published
4. **Bridge technique** -- "That is an interesting question, but what
   is really important here is..."
5. **Silence is an option** -- you do not have to respond to every
   inquiry
6. **Written > Verbal** -- for anything sensitive, communicate in writing
   so there is no misquotation

## Stakeholder Trust Model

### How Trust is Built and Destroyed

```
TRUST EQUATION (Maister):
Trust = (Credibility + Reliability + Intimacy) / Self-Orientation

Credibility:  Do they believe what you say? (expertise, accuracy)
Reliability:  Do you do what you say? (consistency, follow-through)
Intimacy:     Do they feel safe with you? (empathy, discretion)
Self-Orient:  Are you focused on them or yourself? (lower is better)
```

### Trust-Building Behaviors

| Stakeholder | Trust Builder | Trust Destroyer |
|-------------|-------------|----------------|
| Board | No surprises, honest reporting | Hidden bad news, blame |
| Investors | Consistent updates, metrics first | Missed updates, spin |
| Employees | Transparency, follow-through | Saying one thing, doing another |
| Customers | Reliability, responsiveness | Broken promises, poor support |
| Partners | Mutual benefit, consistency | One-sided relationships |
| Press | Honesty, accessibility | Lies, inaccessibility |

## Stakeholder Communication Anti-Patterns

| Anti-Pattern | Description | Fix |
|-------------|------------|-----|
| Board surprise | Board learns major news at the meeting | Pre-brief individually on anything significant |
| Investor ghosting | Stop sending updates when things are bad | Updates are most important when things are bad |
| Employee mushroom | Keep employees in the dark | Default to transparency |
| Customer neglect | Only talk to customers through sales | CEO maintains direct customer relationships |
| Press avoidance | Never engage with media | Proactive media strategy with prepared messages |
| Overpromising | Promise things to every stakeholder | Underpromise, overdeliver |

---

**Stakeholder management is not politics -- it is trust-building. The
CEO Brain maintains trust with every stakeholder group through
consistency, transparency, and authenticity. Trust is the foundation
of every organizational relationship.**
