# Executive Communication -- Written, Verbal, and Visual

## Overview

Communication is the CEO's primary tool. Every strategic decision, every
cultural value, every organizational priority is mediated through
communication. The difference between a good CEO and a great CEO is
often the difference in communication effectiveness.

This module covers the three modes of executive communication -- written
(memos, updates, emails), verbal (all-hands, meetings, 1:1s), and public
(media, conferences, press) -- with frameworks, templates, and practices
for each.

---

## Written Communication

### The CEO Memo

Jeff Bezos banned PowerPoint at Amazon in favor of six-page narrative
memos. The reason: narrative forces clear thinking. Bullets allow
sloppy logic to hide behind formatting.

```
CEO MEMO STRUCTURE (Bezos/Amazon style)
+--------------------------------------------------+
| TITLE: [Clear, specific title]                   |
| DATE: [Date]                                     |
| AUTHOR: [CEO]                                    |
| AUDIENCE: [Who should read this]                 |
|                                                  |
| CONTEXT (1-2 paragraphs):                        |
| Why this memo exists. What happened. What the    |
| situation is. Set the stage.                     |
|                                                  |
| PROBLEM OR OPPORTUNITY (2-3 paragraphs):         |
| What specifically needs to be addressed.         |
| Data and evidence. Customer impact. Business     |
| impact. Why now.                                 |
|                                                  |
| APPROACH (3-4 paragraphs):                       |
| What we are going to do. Why this approach.     |
| What alternatives we considered and rejected.    |
| Key tradeoffs we are making.                    |
|                                                  |
| PLAN (2-3 paragraphs):                          |
| Specific actions. Timeline. Owners. Milestones.  |
| How we will measure success. Kill criteria.      |
|                                                  |
| RISKS (1-2 paragraphs):                         |
| What could go wrong. How we will mitigate.       |
| What we will do if this fails.                  |
+--------------------------------------------------+
```

Memo writing principles:
1. **Write in complete sentences** -- forces complete thoughts
2. **Start with the customer** -- every memo should begin with
   customer impact
3. **Include data** -- assertions must be supported by evidence
4. **Name tradeoffs** -- what are we giving up?
5. **Specify actions** -- who does what by when?
6. **Read aloud** -- if it sounds awkward spoken, rewrite it

### Investor Updates

Monthly investor updates are the CEO's most important written ritual
(see stakeholder_management.md for template).

Additional writing principles for investor updates:
- **Metrics first** -- lead with numbers, not narrative
- **Consistency** -- same format every month (investors compare)
- **Honesty** -- include lowlights alongside highlights
- **Forward-looking** -- what are you focused on next month?
- **Specific asks** -- investors want to help; tell them how

### Internal Email Communication

The CEO's internal emails set the tone for organizational communication.

```
CEO EMAIL FRAMEWORK
+--------------------------------------------------+
| SUBJECT: [Action-oriented, specific]             |
|                                                  |
| OPENING: Why I am writing this (1 sentence)      |
|                                                  |
| CONTEXT: What you need to know (2-3 sentences)   |
|                                                  |
| DECISION/CHANGE: What is happening (clear, direct)|
|                                                  |
| WHY: The reasoning (transparent, honest)          |
|                                                  |
| WHAT IT MEANS FOR YOU: Personal relevance         |
|                                                  |
| NEXT STEPS: What happens now, who does what       |
|                                                  |
| CLOSE: Invitation for questions or feedback       |
+--------------------------------------------------+
```

Email discipline:
- **Write short** -- if it takes more than 2 minutes to read, it is too long
- **Write clearly** -- Hemingway rule: short sentences, simple words
- **One message per email** -- do not bury multiple topics
- **Action in subject line** -- "[Action Required]" or "[FYI]" or "[Decision]"
- **No passive voice** -- "We decided" not "It was decided"
- **No weasel words** -- "We will" not "We might" or "We hope to"

## Verbal Communication

### All-Hands Presentations

The monthly all-hands is the CEO's highest-leverage communication event.
One hour of CEO communication reaches the entire organization.

```
ALL-HANDS DESIGN (60 min)
+--------------------------------------------------+
| OPENING (5 min):                                 |
| - Energy and connection                          |
| - One customer story (grounds everything in      |
|   customer reality)                              |
|                                                  |
| STATE OF THE COMPANY (15 min):                   |
| - Key metrics (3-5 numbers, with trend)         |
| - What is working (celebrate wins)              |
| - What is not working (be honest)               |
| - Why (connect to strategy)                     |
|                                                  |
| STRATEGIC UPDATE (10 min):                       |
| - Where we are going (reinforce vision)         |
| - What is changing (if anything)                |
| - What is NOT changing (stability anchor)       |
|                                                  |
| TEAM SPOTLIGHT (10 min):                         |
| - One team presents their work                  |
| - Builds cross-functional understanding          |
| - Rotates monthly                               |
|                                                  |
| Q&A (20 min):                                   |
| - Open questions (live and pre-submitted)       |
| - CEO answers honestly                          |
| - "I do not know" is a valid answer             |
| - Follow up on unanswered questions within 48h  |
+--------------------------------------------------+
```

### Presentation Principles

1. **Start with why** -- Simon Sinek's framework: why, then how, then what
2. **Tell stories** -- data informs; stories move people
3. **One idea per slide** -- if a slide needs explanation, simplify it
4. **Speak to the back of the room** -- energy, projection, pace
5. **Use silence** -- pauses are powerful; resist filling every moment
6. **End with a call to action** -- what should people DO after this?

### Meeting Communication

The CEO's behavior in meetings defines the meeting culture:

```
CEO MEETING BEHAVIORS
+--------------------------------------------------+
| DO:                                              |
| - Listen before speaking                         |
| - Ask questions before stating conclusions       |
| - Amplify quiet voices ("What do you think?")    |
| - Summarize decisions explicitly                 |
| - Assign actions with owners and deadlines       |
| - End meetings early if the objective is met     |
|                                                  |
| DO NOT:                                          |
| - Dominate the conversation (CEO talk < 30%)     |
| - Shoot down ideas before they are fully stated  |
| - Check your phone (if the CEO does it, everyone |
|   will do it)                                    |
| - Avoid hard topics                              |
| - Allow meetings to run over without explicit    |
|   decision to extend                             |
+--------------------------------------------------+
```

### Difficult Conversations

The CEO must have difficult conversations that others avoid:

```
DIFFICULT CONVERSATION FRAMEWORK
+--------------------------------------------------+
| PREPARE:                                         |
| - What is the specific issue?                   |
| - What is the desired outcome?                  |
| - What are the facts (not interpretations)?      |
| - What is at stake if this is not addressed?    |
|                                                  |
| DELIVER:                                         |
| 1. State the purpose: "I want to discuss..."    |
| 2. Share observations: "What I have seen is..."  |
| 3. State impact: "The impact of this is..."      |
| 4. Listen: "What is your perspective?"           |
| 5. Discuss: Two-way dialogue                     |
| 6. Agree on actions: "Here is what we will do..." |
| 7. Follow up: Check in within 1 week             |
|                                                  |
| PRINCIPLES:                                      |
| - Be direct but not cruel                        |
| - Focus on behavior, not character               |
| - Use specific examples, not generalizations     |
| - Allow the other person to respond              |
| - Agree on specific next steps                   |
+--------------------------------------------------+
```

## Public Speaking and Media

### Conference Presentations

```
CONFERENCE TALK STRUCTURE (20-30 min)
+--------------------------------------------------+
| HOOK (2 min):                                    |
| - Surprising fact, provocative question, or story|
| - Must grab attention in first 30 seconds        |
|                                                  |
| PROBLEM (5 min):                                 |
| - The problem the audience cares about           |
| - Make it personal and relatable                 |
| - Use data to quantify the problem               |
|                                                  |
| INSIGHT (5 min):                                 |
| - Your unique perspective or approach            |
| - The "aha" moment                               |
| - Why this is different from conventional wisdom |
|                                                  |
| EVIDENCE (5-10 min):                             |
| - Stories, data, examples that prove the insight |
| - Customer stories are most compelling           |
| - Be specific (names, numbers, outcomes)         |
|                                                  |
| APPLICATION (5 min):                             |
| - How the audience can apply the insight         |
| - Practical, actionable takeaways               |
| - 3 things to do next                           |
|                                                  |
| CLOSE (2 min):                                   |
| - Callback to the opening hook                   |
| - Memorable final statement                      |
| - Call to action                                 |
+--------------------------------------------------+
```

### Media Interviews

```
MEDIA PREP FRAMEWORK
+--------------------------------------------------+
| BEFORE THE INTERVIEW:                            |
| 1. Define 3 key messages you want to deliver    |
| 2. Anticipate difficult questions (prepare 10)  |
| 3. Prepare transitions to your key messages     |
| 4. Know what you will NOT discuss                |
| 5. Practice with a trusted advisor               |
|                                                  |
| DURING THE INTERVIEW:                            |
| 1. Bridge to your messages: "What is really     |
|    important here is..."                         |
| 2. Use stories and examples (not abstractions)  |
| 3. Be concise (sound bites, not monologues)     |
| 4. Repeat your key message 3 times              |
| 5. If you do not know, say so (do not speculate)|
|                                                  |
| AFTER THE INTERVIEW:                             |
| 1. Offer to fact-check quotes                   |
| 2. Follow up with additional context if needed   |
| 3. Review coverage and learn for next time       |
+--------------------------------------------------+
```

## Delivering Bad News

### The Bad News Framework

Bad news is the ultimate test of CEO communication. The instinct is to
soften, delay, or hide. The discipline is to be direct, early, and honest.

```
BAD NEWS DELIVERY PROTOCOL
+--------------------------------------------------+
| 1. PREPARE:                                      |
|    - Know the full facts                         |
|    - Have a plan for addressing the issue        |
|    - Anticipate questions and emotions           |
|                                                  |
| 2. DELIVER PROMPTLY:                             |
|    - "I have difficult news to share."           |
|    - State the news clearly and directly         |
|    - Do not bury it in context or qualifications |
|                                                  |
| 3. OWN IT:                                       |
|    - "This is my responsibility."                |
|    - Do not blame others                         |
|    - Do not minimize                             |
|                                                  |
| 4. EXPLAIN:                                      |
|    - What happened and why                       |
|    - What we are doing about it                  |
|    - What we will do differently going forward   |
|                                                  |
| 5. EMPATHIZE:                                    |
|    - Acknowledge the impact on people            |
|    - Allow emotional response                    |
|    - Be present (do not rush through it)         |
|                                                  |
| 6. SUPPORT:                                      |
|    - What support is available                   |
|    - What happens next                           |
|    - How to get questions answered               |
+--------------------------------------------------+
```

### Types of Bad News and Approach

| Bad News | Audience | Key Principle |
|----------|----------|--------------|
| Layoffs | Employees | Dignity, support, honesty about why |
| Missed targets | Board/investors | Honest, with root cause and plan |
| Product failure | Customers | Empathy, accountability, remediation |
| Executive departure | All | Quick, factual, forward-looking |
| Pivot | Employees | Vision for the new direction, why |
| Funding failure | Team | Honest about runway, options |

## Communication Style Guide

### CEO Writing Voice

```
BE:                          DO NOT BE:
- Direct                     - Vague
- Honest                     - Spin-heavy
- Concise                    - Verbose
- Forward-looking            - Backward-dwelling
- Empathetic                 - Cold
- Specific                   - Abstract
- Confident                  - Arrogant
- Humble when wrong          - Defensive
```

### The Hemingway Test

Before sending any communication, apply:
1. Can any sentence be shorter? Shorten it.
2. Can any word be simpler? Simplify it.
3. Can any paragraph be cut? Cut it.
4. Is the main point in the first paragraph? Move it there.
5. Would a new employee understand this? If not, clarify.
