# Global Content Strategy — Authoritative Module

Global content strategy governs how content is created, structured, and
managed to maximize translatability, minimize localization cost, and
ensure cultural effectiveness across all target markets. Source content
quality is the single largest determinant of translation quality and cost.
This document codifies authoring guidelines, content governance, content
types, and multimedia localization strategy.

---

## 1. WRITING FOR TRANSLATION

### Controlled Language Principles

Controlled language reduces ambiguity, shortens sentences, and
eliminates idioms — dramatically improving translation quality and
reducing cost.

| Principle | Example (Bad) | Example (Good) |
|-----------|--------------|----------------|
| Use short sentences | "When you've finished entering your data and are ready to move on, click the button at the bottom." | "Enter your data. Click Submit." |
| Avoid idioms | "This feature is a game-changer." | "This feature significantly improves workflow." |
| Use active voice | "The report was generated by the system." | "The system generated the report." |
| One meaning per word | "Save your changes." / "Save money." | Use "Save" only for one context; use "Reduce cost" for other |
| Avoid humor/wordplay | "Lettuce help you eat better!" | "We help you eat better." |
| Avoid cultural references | "It's a home run!" | "It's a major success." |
| Use consistent terminology | "Account" / "Profile" / "User page" | Always "Account" for the same concept |
| Specify referents | "Click it to continue." | "Click the Submit button to continue." |
| Avoid noun strings | "System error message display settings" | "Settings for displaying system error messages" |
| Use articles | "Click button." | "Click the button." |

### Source Language Quality Checklist

```
Before sending content for translation:
├── [ ] Spell-checked and grammar-reviewed
├── [ ] Sentences average <20 words
├── [ ] No idioms, slang, or colloquialisms
├── [ ] No cultural references (sports, holidays, food)
├── [ ] Active voice used throughout
├── [ ] Consistent terminology (glossary-compliant)
├── [ ] Variables/placeholders have descriptions
├── [ ] Character limits documented for UI strings
├── [ ] Context provided (screenshots, descriptions)
├── [ ] Content type identified (UI, marketing, legal, help)
└── [ ] Reviewed by someone other than the author
```

## 2. CONTENT TYPES AND LOCALIZATION DEPTH

### Content Type Matrix

| Content Type | L10n Depth | Quality Level | Typical Approach |
|-------------|-----------|---------------|-----------------|
| UI strings | Full | Highest | Human translation + review |
| Onboarding | Full | Highest | Human + cultural adaptation |
| Error messages | Full | High | Human translation |
| Help/documentation | Selective | High | MTPE for volume |
| Marketing copy | Full + adapt | Highest | Transcreation |
| Blog posts | Selective | Medium | MTPE or skip |
| Legal/compliance | Full | Highest | Certified translation |
| Email templates | Full | High | Human translation |
| Push notifications | Full | High | Human (short, impactful) |
| Knowledge base | Selective | Medium | MTPE + community |
| Release notes | Selective | Medium | MT + light PE |
| Social media | Adapt | High | In-market creation preferred |
| Video scripts | Full | Highest | Human + voiceover/subtitle |
| API docs | Selective | Medium | MTPE or English-only |

### Content Prioritization for Translation

```
Priority 1 (Always translate):
├── User-facing UI strings
├── Onboarding and activation flows
├── Payment and billing content
├── Error messages (critical path)
├── Legal requirements (privacy, ToS)
└── Core marketing (landing page, CTAs)

Priority 2 (Translate for Tier 1-2 locales):
├── Help center articles (top 20 by views)
├── Email templates (lifecycle)
├── In-app guides and tooltips
├── Feature announcements
└── Customer-facing documentation

Priority 3 (Translate selectively):
├── Blog posts (evergreen only)
├── Knowledge base (remainder)
├── Release notes
├── Developer documentation
└── Community content

Priority 4 (English-only or MT):
├── Internal documentation
├── API reference
├── Changelog (technical)
├── Forum/community (user-generated)
└── Debug/developer messages
```

## 3. CONTENT GOVERNANCE

### Terminology Management

| Component | Description | Owner |
|-----------|-------------|-------|
| Product glossary | Canonical terms for product concepts | Product + L10n |
| Do-Not-Translate list | Brand names, product names, technical terms | L10n |
| Style guide per locale | Tone, formality, conventions per language | L10n + linguists |
| Forbidden terms | Terms that must never be used | Legal + L10n |
| Preferred terms | Terms that should always be used | Product + L10n |

### Content Review Workflow

```
Content Author Creates Source
    │
    ▼
L10n Review (translatability check)
├── Controlled language compliance?
├── Terminology glossary-compliant?
├── Context provided?
├── Character limits defined?
└── Cultural neutrality verified?
    │
    ▼ (If issues found)
Return to Author with Feedback
    │
    ▼ (If approved)
Content Enters Translation Pipeline
```

### Source Content Change Management

| Change Type | Impact | Protocol |
|------------|--------|----------|
| New content | New translation needed | Standard workflow |
| Minor edit (typo fix) | May not need retranslation | Flag, leverage TM |
| Moderate edit (wording) | Retranslation needed | Push to TMS, use TM leverage |
| Major rewrite | Full retranslation | Treat as new content |
| Content removal | Remove translations | Clean up all locale files |
| Key rename | No retranslation needed | Update key mapping in TMS |

## 4. SEO LOCALIZATION

### International SEO Strategy

| Element | Approach | Common Mistakes |
|---------|----------|----------------|
| URL structure | Subdirectory (`/de/`) or subdomain (`de.example.com`) | ccTLD per market (expensive, complex) |
| hreflang tags | Implement for all locale variants | Missing, incorrect, or incomplete tags |
| Keyword research | Per-market research (not translation) | Translating EN keywords literally |
| Meta titles/descriptions | Localized per market | Auto-translated, no keyword optimization |
| Content parity | Equivalent content per locale | Copy of EN structure without adaptation |
| Internal linking | Locale-consistent linking | Cross-locale internal links |
| Sitemap | Per-locale sitemaps | Single sitemap without locale info |

### Keyword Localization Process

```
1. Identify top-performing EN keywords
2. Research equivalent terms per target market
    ├── NOT direct translation
    ├── Use local search data (Google Keyword Planner per market)
    ├── Analyze local competitor keywords
    └── Validate with native speaker
3. Map localized keywords to content
4. Optimize meta tags, headings, body content
5. Monitor ranking per locale
6. Iterate based on performance data
```

### hreflang Implementation

```html
<!-- On English page -->
<link rel="alternate" hreflang="en" href="https://example.com/en/page" />
<link rel="alternate" hreflang="de" href="https://example.com/de/page" />
<link rel="alternate" hreflang="ja" href="https://example.com/ja/page" />
<link rel="alternate" hreflang="x-default" href="https://example.com/en/page" />
```

## 5. MULTIMEDIA LOCALIZATION

### Video Localization Options

| Method | Cost | Quality | Turnaround | Best For |
|--------|------|---------|------------|----------|
| Subtitling | Low | Good | Fast | Informational, tutorials |
| Voiceover | Medium | Good | Medium | E-learning, presentations |
| Dubbing | High | Excellent | Slow | Marketing, brand videos |
| AI voiceover | Low | Improving | Fast | Internal, low-stakes |
| Re-creation | Highest | Best | Slowest | Key marketing assets |

### Subtitling Guidelines

| Parameter | Standard |
|-----------|----------|
| Max characters per line | 42 (single line), 84 (two lines) |
| Max lines | 2 |
| Min display time | 1 second |
| Max display time | 7 seconds |
| Reading speed | 150-180 words/minute |
| Font | Sans-serif, high contrast |
| Position | Bottom center, above lower third |

### Image Localization

| Image Type | Localization Need | Approach |
|-----------|-------------------|----------|
| Screenshots | Retake in target locale | Automated screenshot pipeline |
| Illustrations with text | Recreate with translated text | Editable source files required |
| Icons with cultural meaning | Adapt or replace | Cultural review |
| Stock photos with people | May need demographic adaptation | Region-specific stock |
| Infographics | Full localization | Layered design files required |
| Logos with text | Do NOT translate unless brand requires | Brand guidelines |

## 6. CONTENT MANAGEMENT SYSTEM (CMS) L10N

### CMS Requirements for Localization

| Requirement | Description |
|------------|-------------|
| Multi-locale support | Manage content in multiple locales natively |
| Locale fallback | Display source language if translation missing |
| Field-level translation | Translate individual fields, not just pages |
| Translation workflow | Assign, track, review within CMS |
| TMS integration | Connect to external TMS via API/webhook |
| Preview per locale | Preview content in each locale before publish |
| Versioning per locale | Track changes independently per locale |
| RTL support | Render RTL content correctly |

### CMS + TMS Integration Pattern

```
CMS (content authoring)
    │
    ▼ (webhook: content updated)
Connector / Middleware
    │
    ▼ (API: push source content)
TMS (translation management)
    │
    ▼ (webhook: translation complete)
Connector / Middleware
    │
    ▼ (API: update translated content)
CMS (publish localized content)
```

## 7. USER-GENERATED CONTENT (UGC) STRATEGY

### UGC Localization Options

| Approach | Quality | Cost | Use Case |
|----------|---------|------|----------|
| No translation | N/A | Free | Community stays in original language |
| MT auto-translate | Variable | Low | Display with "auto-translated" label |
| Community translation | Variable | Free | Volunteer translators |
| Professional translation | High | High | High-value UGC (testimonials, reviews) |

### UGC Decision Framework

```
Is the UGC high-value (testimonial, case study)?
├── Yes → Professional translation
└── No → Is it user-facing in a critical flow?
    ├── Yes → MT with "auto-translated" disclaimer
    └── No → Leave in original language
```

## 9. ANTI-PATTERNS

| Anti-Pattern | Description | Fix |
|-------------|-------------|-----|
| Translate everything | All content translated regardless of value | Prioritize by impact |
| Source language poor quality | Ambiguous, idiomatic source text | Controlled language review |
| No context for translators | Strings without screenshots or descriptions | Context provision system |
| Keyword translation | Translating SEO keywords literally | Per-market keyword research |
| Subtitle = translation | Treating subtitles as pure translation | Subtitle-specific guidelines |
| CMS not l10n-ready | CMS cannot manage multilingual content | Evaluate CMS for l10n needs |
| Ignore UGC | No strategy for user-generated content | UGC decision framework |

---

**Global content strategy is the upstream lever that determines downstream
localization quality, cost, and speed. Every improvement to source content
quality reduces translation cost, improves linguistic quality, and
accelerates time-to-market for every locale. The best localization teams
invest as much in source content governance as they do in translation
management — because the source is where quality begins.**
