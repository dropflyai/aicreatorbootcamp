# Branding Review Checklist — Quality Gate (Authoritative)

This checklist must be completed before any branding deliverable is approved.
Every section must pass. Partial completion is a fail.

If a checklist item cannot be verified, it has not been done.

---

## HOW TO USE THIS CHECKLIST

1. Complete every section for the branding deliverable under review
2. Mark each item: PASS, FAIL, or N/A (with justification for N/A)
3. All items must be PASS or justified N/A
4. Any FAIL = deliverable is not approved
5. Reviewer must sign off with name, date, and verdict
6. Attach evidence (mockups, research, audit data) for every PASS claim

---

## SECTION 1: BRAND STRATEGY FOUNDATION

### 1.1 Brand Purpose and Positioning
- [ ] Brand purpose is documented and answers "Why does this brand exist beyond profit?"
- [ ] Brand positioning statement follows the format: For [audience], [brand] is the [category] that [key benefit] because [reason to believe]
- [ ] Positioning is distinctive — does not apply to any competitor when you swap the brand name
- [ ] Positioning is defensible — the brand can credibly deliver on the promise
- [ ] Positioning is desirable — the target audience cares about the claimed benefit
- [ ] Brand values are defined (3-5 maximum) and are actionable (not generic)
- [ ] Brand values are distinguishable from generic values (e.g., "integrity" and "innovation" are too generic unless defined specifically)

### 1.2 Target Audience
- [ ] Primary target audience is defined with demographic, psychographic, and behavioral data
- [ ] Secondary audiences are identified and prioritized
- [ ] Audience segments are validated with research (not assumptions)
- [ ] Audience pain points and aspirations are documented
- [ ] Audience language and communication preferences are understood
- [ ] Audience brand relationship expectations are mapped

### 1.3 Competitive Landscape
- [ ] Direct competitors are identified and analyzed (minimum 5)
- [ ] Indirect competitors and substitutes are considered
- [ ] Competitive positioning map is created showing white space
- [ ] Competitor brand strengths and vulnerabilities are documented
- [ ] Category conventions are identified (what to follow vs. break)
- [ ] Differentiation strategy is articulated and defensible

### 1.4 Brand Promise
- [ ] Brand promise is articulated in one sentence
- [ ] Promise is specific (not "best" or "innovative")
- [ ] Promise is measurable (you can tell if it is being kept)
- [ ] Promise aligns with operational capability (can the company actually deliver?)
- [ ] Promise resonates with the target audience (validated, not assumed)

## SECTION 2: VISUAL IDENTITY

### 2.1 Logo System
- [ ] Primary logo is designed and finalized
- [ ] Logo variations exist for different contexts (full, abbreviated, icon-only)
- [ ] Clear space rules are defined and documented
- [ ] Minimum size requirements are established
- [ ] Logo works on both light and dark backgrounds
- [ ] Logo is legible at small sizes (favicon, app icon)
- [ ] Logo file formats are complete (SVG, PNG, EPS at minimum)
- [ ] Logo misuse examples are documented (what NOT to do)

### 2.2 Color System
- [ ] Primary color palette is defined with hex, RGB, CMYK, and Pantone values
- [ ] Secondary/accent colors are defined
- [ ] Color ratios are specified (e.g., 60/30/10 rule)
- [ ] Color accessibility is verified (WCAG AA minimum for text colors)
- [ ] Color meanings are documented (what each color communicates)
- [ ] Dark mode color system is defined (if applicable)
- [ ] Color usage rules are documented (when to use which color)

### 2.3 Typography
- [ ] Primary typeface is selected with rationale
- [ ] Secondary typeface is selected (if needed) with pairing rationale
- [ ] Type scale is defined (heading levels, body, captions, labels)
- [ ] Font weights used are specified (not the entire font family)
- [ ] Line height and letter spacing guidelines are established
- [ ] Web font licensing is secured
- [ ] Fallback fonts are specified
- [ ] Typography hierarchy creates clear information structure

### 2.4 Imagery and Photography
- [ ] Photography style is defined (subjects, composition, lighting, mood)
- [ ] Illustration style is defined (if applicable)
- [ ] Iconography style is defined and consistent
- [ ] Image quality standards are documented
- [ ] Image sourcing guidelines exist (stock photography rules)
- [ ] Image diversity and representation guidelines are established
- [ ] Image treatment rules (filters, overlays, crops) are documented

### 2.5 Layout and Composition
- [ ] Grid system is defined
- [ ] Spacing system is tokenized (consistent spacing scale)
- [ ] Component-level design principles are documented
- [ ] Responsive behavior is specified for key breakpoints
- [ ] White space usage guidelines exist
- [ ] Visual hierarchy principles are documented with examples

**Section Verdict:** PASS / FAIL
**Evidence attached:** [ ]

## SECTION 3: VERBAL IDENTITY

### 3.1 Brand Voice
- [ ] Brand voice is defined with 3-5 personality traits
- [ ] Each trait has a spectrum (e.g., "Confident, not arrogant")
- [ ] Voice guide includes do's and don'ts with examples
- [ ] Voice adapts for context while remaining recognizable (examples provided for: formal, casual, urgent, celebratory)
- [ ] Voice is authentic to company culture (not aspirational fiction)
- [ ] Voice guide has been tested with content creators for usability

### 3.2 Tone Guidelines
- [ ] Tone variations are defined for different situations
- [ ] Error and apology tone is specifically addressed
- [ ] Sales and marketing tone guidelines exist
- [ ] Support and service tone guidelines exist
- [ ] Internal communication tone guidelines exist
- [ ] Tone adjustments for different audience segments are documented

### 3.3 Messaging Framework
- [ ] Key messages are defined for each audience segment
- [ ] Value propositions are articulated for each product/service
- [ ] Elevator pitch exists (10-second, 30-second, 2-minute versions)
- [ ] Tagline or brand line is developed (if applicable)
- [ ] Proof points support each key message
- [ ] Message hierarchy is clear (what to lead with, what to support with)

### 3.4 Naming Conventions
- [ ] Naming principles are documented
- [ ] Product/feature naming conventions are established
- [ ] Naming hierarchy aligns with brand architecture
- [ ] Names are checked for trademark availability (preliminary)
- [ ] Names are checked for linguistic/cultural issues in target markets
- [ ] Naming process for new products/features is documented

### 3.5 Content Standards
- [ ] Writing style guide exists (grammar, punctuation, formatting preferences)
- [ ] Jargon and terminology glossary is maintained
- [ ] Inclusive language guidelines are established
- [ ] SEO writing guidelines exist (if applicable)
- [ ] Legal/compliance copy requirements are documented
- [ ] Content approval workflow is defined

**Section Verdict:** PASS / FAIL
**Evidence attached:** [ ]

## SECTION 4: BRAND EXPERIENCE

### 4.1 Touchpoint Mapping
- [ ] All brand touchpoints are identified and catalogued
- [ ] Touchpoints are prioritized by importance and frequency
- [ ] Brand expression is defined for each touchpoint
- [ ] Touchpoint ownership is assigned (who is responsible for brand quality)
- [ ] Touchpoint experience is consistent with brand promise

### 4.2 Customer Journey Brand Integration
- [ ] Brand expression is mapped to each stage of the customer journey
- [ ] Key moments of truth are identified and brand-optimized
- [ ] Transitions between touchpoints maintain brand continuity
- [ ] Emotional arc of the customer journey is designed
- [ ] Pain points in the journey are addressed with brand-aligned solutions

### 4.3 Digital Brand Experience
- [ ] Website reflects brand identity consistently
- [ ] Social media profiles are brand-consistent across platforms
- [ ] Email communications follow brand guidelines
- [ ] Product/app UI reflects brand identity
- [ ] Digital advertising follows brand guidelines
- [ ] Chatbot or automated communications are in brand voice

### 4.4 Physical Brand Experience (if applicable)
- [ ] Physical spaces reflect brand identity
- [ ] Packaging follows brand guidelines
- [ ] Print materials follow brand guidelines
- [ ] Merchandise or swag is on-brand
- [ ] Signage and wayfinding are brand-consistent
- [ ] Staff presentation aligns with brand image

### 4.5 Signature Moments
- [ ] At least one "signature moment" is designed into the brand experience
- [ ] Signature moment is unique to this brand (competitors do not do this)
- [ ] Signature moment reinforces the brand promise
- [ ] Signature moment is repeatable and scalable
- [ ] Signature moment is measurable (can track if it happens and its impact)

**Section Verdict:** PASS / FAIL
**Evidence attached:** [ ]

## SECTION 5: BRAND GOVERNANCE

### 5.1 Brand Guidelines Document
- [ ] Comprehensive brand guidelines document exists
- [ ] Guidelines are accessible to all team members and partners
- [ ] Guidelines are versioned with change log
- [ ] Guidelines include enough examples to be self-service
- [ ] Guidelines distinguish between rules (must follow) and recommendations (should follow)
- [ ] Guidelines are written for non-designers (not just the design team)

### 5.2 Brand Asset Management
- [ ] Brand assets are stored in a centralized, accessible location
- [ ] Asset library is organized and searchable
- [ ] Asset versions are controlled (no outdated logos floating around)
- [ ] Asset usage rights and licensing are tracked
- [ ] Templates exist for common use cases (presentations, social posts, email)
- [ ] Self-service asset access is available for authorized users

### 5.3 Brand Compliance Process
- [ ] Brand review process exists for new materials
- [ ] Review authority is defined (who can approve brand usage)
- [ ] Compliance monitoring is active (periodic audits)
- [ ] Non-compliance has defined consequences and remediation
- [ ] Brand compliance training is available for new team members
- [ ] Partner and vendor brand compliance is contractually required

### 5.4 Brand Evolution Process
- [ ] Process exists for proposing brand changes
- [ ] Brand changes are evaluated against brand equity impact
- [ ] Brand refresh triggers are defined (when is a refresh needed)
- [ ] Brand evolution vs. revolution criteria are documented
- [ ] Historical brand decisions are archived for institutional memory

**Section Verdict:** PASS / FAIL
**Evidence attached:** [ ]

## SECTION 6: BRAND MEASUREMENT

### 6.1 Brand Health Metrics
- [ ] Brand awareness is tracked (aided and unaided)
- [ ] Brand perception is measured regularly
- [ ] Brand sentiment is monitored (social listening, review analysis)
- [ ] Brand preference vs. competitors is measured
- [ ] Brand metrics have baselines and targets

### 6.2 Brand Performance Metrics
- [ ] Brand's impact on business metrics is measured (conversion, retention, premium)
- [ ] Brand-attributed revenue or pipeline is tracked
- [ ] Brand investment ROI is calculated
- [ ] Brand performance is benchmarked against competitors
- [ ] Metrics are reported at regular cadence to stakeholders

### 6.3 Brand Research Program
- [ ] Regular brand research is scheduled (not just ad hoc)
- [ ] Research methodology is consistent for trend comparison
- [ ] Research includes both quantitative and qualitative methods
- [ ] Research findings are actionable (not just informational)
- [ ] Research informs brand strategy decisions with documented links

**Section Verdict:** PASS / FAIL
**Evidence attached:** [ ]

## SECTION 7: BRAND PROTECTION

### 7.1 Legal Protection
- [ ] Trademark registration is filed or completed for brand name
- [ ] Trademark registration for logo (if applicable)
- [ ] Domain names are secured (primary + variations)
- [ ] Social media handles are secured across major platforms
- [ ] Trademark monitoring is active for potential infringement
- [ ] Brand usage terms are included in partner/vendor contracts

### 7.2 Reputation Protection
- [ ] Brand reputation monitoring is active
- [ ] Crisis communication plan exists for brand-threatening events
- [ ] Spokesperson and communication protocols are defined
- [ ] Social media crisis response guidelines exist
- [ ] Brand recovery playbook exists for post-crisis situations
