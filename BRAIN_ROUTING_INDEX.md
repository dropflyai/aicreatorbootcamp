## Common Task Routing

### "Build a new feature"
```
1. product_brain → Define requirements
2. design_brain → Create UI/UX
3. engineering_brain → Implement (may spawn: backend, frontend, database)
4. qa_brain → Test
```

### "Fix a bug"
```
1. debugger_brain → Diagnose and fix
2. qa_brain → Verify fix
```

### "Launch a new product"
```
1. research_brain → Market analysis
2. mba_brain → Business model
3. product_brain → Requirements
4. design_brain + engineering_brain → Build
5. marketing_brain → Go-to-market
6. sales_brain → Sales playbook
```

### "Improve performance"
```
1. performance_brain → Profile and identify bottlenecks
2. backend_brain or frontend_brain → Implement optimizations
3. qa_brain → Verify no regressions
```

### "Add AI capabilities"
```
1. ai_brain → Strategy and model selection
2. data_brain → Data pipelines
3. engineering_brain → Integration
4. security_brain → Review for AI safety
```

### "Scale infrastructure"
```
1. architecture_brain → Design
2. cloud_brain → Infrastructure
3. devops_brain → Deployment
4. security_brain → Security review
```

### "Raise funding"
```
1. finance_brain → Financial model
2. investor_brain → Pitch deck and strategy
3. mba_brain → Business narrative
```

### "Enter new market"
```
1. research_brain → Market analysis
2. localization_brain → Adaptation requirements
3. legal_brain → Compliance
4. marketing_brain → Positioning
```

## Multi-Brain Patterns

### Parallel Audit (Used in VoiceFly)
Spawn 5 agents simultaneously:
```
├── backend_brain → API routes, server logic
├── database_brain → Schema, migrations, RLS
├── frontend_brain → UI components, state, flows
├── security_brain → Auth, vulnerabilities
└── qa_brain → Test coverage, error handling
```

### New Project Research
Spawn simultaneously:
```
├── research_brain → Industry analysis
├── mba_brain → Business model
├── marketing_brain → Target demographics
├── finance_brain → Cost/revenue model
├── product_brain → MVP requirements
└── legal_brain → Compliance requirements
```

### Technical Deep-Dive
Spawn based on stack:
```
├── architecture_brain → Overall design
├── backend_brain → API layer
├── frontend_brain → UI layer
├── database_brain → Data layer
├── devops_brain → Infrastructure
└── security_brain → Security review
```

## Decision: Which Brain?

**Ask yourself:**

1. **Is this about WHAT to build?** → product_brain
2. **Is this about HOW to build?** → engineering_brain (or specialist)
3. **Is this about HOW it looks?** → design_brain
4. **Is this about WHO buys?** → marketing_brain or sales_brain
5. **Is this about MONEY?** → finance_brain or pricing_brain
6. **Is this about RISK?** → legal_brain or security_brain
7. **Is this about GROWTH?** → growth_brain or partnership_brain
8. **Is this about KEEPING customers?** → customer_success_brain or support_brain
9. **Is this about DATA?** → data_brain or analytics_brain
10. **Is this about AI?** → ai_brain

**When in doubt:** Route through ceo_brain. It will decompose and delegate.

## How Brains Communicate

Brains don't call each other programmatically. Instead:

1. **CEO Brain** receives the task
2. **CEO Brain** decomposes into brain-specific subtasks
3. **Claude** spawns parallel Task agents, each following a brain's protocols
4. **Results** are synthesized
5. **Learnings** are logged to memory

The "wiring" is documentation + Claude's orchestration, not code.

See: `/infrastructure/BRAIN_WIRING_MAP.md` for full communication protocols.
