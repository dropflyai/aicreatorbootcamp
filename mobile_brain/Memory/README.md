# Mobile Brain — Memory

## Purpose

This file serves as the institutional memory of the Mobile Brain. It
records decisions made, lessons learned, patterns discovered, and mistakes
to avoid. When the Mobile Brain encounters a similar situation in the
future, it consults this memory before making recommendations.

---

## Format

Each entry follows this structure:

```
### [YYYY-MM-DD] — [Brief Title]

**Context**: What was the situation?
**Decision**: What was decided?
**Outcome**: What happened?
**Lesson**: What should we remember?
**Tags**: [architecture | performance | security | distribution | etc.]
```

---

## Memory Categories

| Category | Description |
|----------|------------|
| architecture | Architectural decisions and their outcomes |
| performance | Performance optimizations and their measured impact |
| security | Security decisions and vulnerability resolutions |
| distribution | App Store/Play Store submission experiences |
| platform | Platform-specific behaviors and gotchas |
| tooling | Build tools, CI/CD, and developer experience |
| patterns | New patterns discovered or existing patterns evolved |
| failures | Things that went wrong and why |

---

## Memory Rules

1. **Record every significant decision** — Future instances of this brain
   will benefit from knowing why a choice was made.
2. **Record every surprise** — If platform behavior was unexpected, it
   will be unexpected again.
3. **Record every failure** — The most valuable entries are the ones that
   prevent repeating mistakes.
4. **Keep entries concise** — A long entry is an unread entry.
5. **Tag consistently** — Tags enable fast lookup by category.

---

**Memory is what separates experienced engineers from beginners. Record everything.**
