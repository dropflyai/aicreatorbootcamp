# Ideation Methods -- Systematic Creativity

## Overview

Ideation is the structured generation of new ideas, concepts, and opportunities.
Contrary to the romantic myth of the lone genius, research consistently shows that
systematic ideation methods outperform unstructured brainstorming. The best ideas
emerge from disciplined processes that combine divergent thinking (generating options)
with convergent thinking (evaluating and selecting). This module catalogs the most
effective ideation methods, grounded in empirical research on creative performance.

## Design Thinking (Stanford d.school)

### The Five Stages

```
+----------+     +----------+     +----------+     +-----------+     +----------+
| EMPATHIZE| --> |  DEFINE  | --> |  IDEATE  | --> | PROTOTYPE | --> |   TEST   |
| Understand|    | Frame the|     | Generate |     | Build to  |     | Learn    |
| the human |    | right    |     | radical  |     | think     |     | from     |
| problem   |    | problem  |     | solutions|     |           |     | users    |
+----------+     +----------+     +----------+     +-----------+     +----------+
     |                                                                      |
     +----------------------------------------------------------------------+
                         Iterate based on learning
```

### Stage 1: Empathize

**Purpose**: Develop deep understanding of people and their needs.

Methods:
- **Contextual inquiry**: Observe users in their natural environment
- **Empathy interviews**: Open-ended, probing conversations (not surveys)
- **Journey mapping**: Document the full user experience over time
- **Shadowing**: Follow users through their day
- **Extreme user research**: Study lead users and edge cases

Rules:
- Assume you know nothing
- Observe behavior, not just stated preferences
- Look for workarounds (signals of unmet needs)
- Capture emotions, frustrations, and delights
- Document with quotes, photos, and artifacts

### Stage 2: Define

**Purpose**: Synthesize empathy findings into actionable problem statements.

Key tool -- Point of View (POV) statement:
```
[USER] needs to [NEED] because [INSIGHT].
```

Example: "Busy working parents need a way to prepare healthy meals in under 15
minutes because they feel guilty about feeding their children processed food but
have no time after work."

### Stage 3: Ideate

**Purpose**: Generate the widest possible range of solutions.

This is where the ideation methods below are applied. The goal is quantity and
diversity, not quality. Evaluation comes later.

### Stage 4: Prototype

**Purpose**: Build to think. Make ideas tangible to test assumptions.

Fidelity spectrum: sketch --> paper prototype --> wireframe --> clickable --> functional

### Stage 5: Test

**Purpose**: Test prototypes with real users to validate or invalidate assumptions.

Rules:
- Test the solution, not the user
- Let the user drive the interaction
- Watch for confusion, delight, and workarounds
- Ask "why" five times to get to root insights

## Brainstorming -- What Actually Works

### Osborn's Original Rules (1953)

Alex Osborn established four rules for brainstorming:
1. **Defer judgment**: No criticism during idea generation
2. **Go for quantity**: More ideas increase chances of finding great ones
3. **Build on others' ideas**: "Yes, and..." not "No, but..."
4. **Encourage wild ideas**: Unconventional ideas often lead to breakthroughs

### What Modern Research Shows

Research by Diehl & Stroebe (1987), Paulus & Nijstad (2003), and others reveals
that traditional group brainstorming underperforms individual ideation due to:

| Problem | Mechanism | Solution |
|---------|-----------|---------|
| Production blocking | Only one person speaks at a time | Brainwriting, silent generation first |
| Evaluation apprehension | Fear of judgment despite rules | Anonymous submission, psychological safety |
| Social loafing | Reduced effort in groups | Individual accountability, smaller groups |
| Anchoring | First idea anchors all subsequent ideas | Independent generation before sharing |
| Conformity | Converging on shared ideas | Diverse teams, explicit diverse prompts |

### Evidence-Based Brainstorming Protocol

```
Phase 1: Individual Ideation (10 min)
  - Each person generates ideas silently in writing
  - No discussion, no sharing
  - Use prompts (SCAMPER, How Might We, analogies)

Phase 2: Share and Build (15 min)
  - Round-robin sharing (each person shares one idea)
  - Others write additions/variations (not verbal critique)
  - No evaluation, only building

Phase 3: Clustering (10 min)
  - Group similar ideas into themes
  - Name each cluster
  - Identify gaps (areas with few ideas)

Phase 4: Gap Filling (10 min)
  - Target ideation for under-represented clusters
  - Use specific prompts for weak areas

Phase 5: Dot Voting (5 min)
  - Each person gets 3-5 votes
  - Vote for ideas with most potential (not most feasible)
  - Tally and discuss top ideas
```

## SCAMPER Method (Eberle, 1971)

Systematic prompts for modifying existing products, services, or processes:

| Letter | Prompt | Example Questions |
|--------|--------|-------------------|
| **S**ubstitute | What can be replaced? | Different materials, people, processes, approaches? |
| **C**ombine | What can be merged? | Combine features, purposes, audiences, technologies? |
| **A**dapt | What can be borrowed? | What else is like this? What analogies apply? |
| **M**odify | What can be changed? | Change size, shape, color, frequency, meaning? |
| **P**ut to other use | What else could it do? | New use cases, different markets, different contexts? |
| **E**liminate | What can be removed? | Remove features, steps, constraints, assumptions? |
| **R**everse/Rearrange | What can be flipped? | Reverse order, swap roles, invert assumptions? |

### SCAMPER Application Process

```
1. Define the product/service/process to innovate
2. Walk through each SCAMPER prompt systematically
3. Generate 3-5 ideas per prompt (minimum)
4. Total: 21-35 ideas in a single session
5. Cluster and evaluate using impact-effort matrix
```

## How Might We (HMW) Questions

### The HMW Format

Transform insights and problems into opportunity questions:

```
Problem:  "Users abandon the checkout because it has too many steps."
HMW:      "How might we make checkout feel effortless?"
          "How might we eliminate checkout entirely?"
          "How might we make each step feel valuable?"
          "How might we let users checkout in one action?"
```

### HMW Generation Rules

1. **Not too broad**: "How might we fix everything?" (useless)
2. **Not too narrow**: "How might we add a skip button?" (already a solution)
3. **Just right**: "How might we reduce friction at the moment of purchase?"

### HMW Variations

| Variation | Pattern | Example |
|-----------|---------|---------|
| Amplify the good | HMW amplify [positive aspect]? | HMW make the waiting time feel valuable? |
| Remove the bad | HMW eliminate [negative aspect]? | HMW eliminate the need to wait? |
| Explore the opposite | HMW make [opposite] happen? | HMW make users want to wait longer? |
| Question assumptions | HMW challenge [assumption]? | HMW serve users without an account? |
| Change status quo | HMW change [existing norm]? | HMW make returns more exciting than buying? |
| Analogize | HMW make it like [analogy]? | HMW make checkout like a concierge experience? |

## Analogical Thinking

### The Power of Cross-Domain Transfer

Many breakthrough innovations come from applying solutions from one domain to
problems in another:

| Innovation | Source Domain | Target Domain |
|-----------|-------------|---------------|
| Velcro | Burdock burrs (nature) | Fastening technology |
| Assembly line | Meatpacking plants | Automobile manufacturing |
| Airbnb | Craigslist, couchsurfing | Hospitality industry |
| Spotify | Netflix (streaming model) | Music industry |

### Analogical Thinking Process

```
1. Abstract the problem to its essential structure
   "We need to [function] for [user] in [context]"

2. Search for analogous domains that solve similar structural problems
   "Who else needs to [similar function]?"

3. Map the solution from the source domain to the target domain
   "How does [source domain] solve this, and what translates?"

4. Adapt the mapped solution to the specific constraints of the target
   "What modifications are needed for our context?"
```

## 10x Thinking (Moonshot Thinking)

### The Google X Principle

Instead of improving by 10%, aim to improve by 10x. This forces fundamentally
different approaches rather than incremental improvements.

```
10% improvement:
  - Optimize existing approach
  - Squeeze more from current solution
  - Incremental, safe, predictable
  - Competition: doing the same thing slightly better

10x improvement:
  - Start from first principles
  - Question every assumption
  - Requires fundamentally new approach
  - Competition: doing something entirely different
```

### First Principles Reasoning (Musk)

```
Step 1: Identify the fundamental truths (first principles)
        "What do we know to be true, stripped of all assumptions?"

Step 2: Question every assumption built on those principles
        "Why do we do it this way? Is that a law of physics or a convention?"

Step 3: Build up from first principles to a new solution
        "What is the best possible solution if we start from scratch?"
```

Example: SpaceX rockets
- Convention: Rockets are disposable (assumed truth)
- First principle: There is no physics reason rockets cannot land
- Innovation: Reusable rockets (10x cost reduction)

## Jobs-to-Be-Done Opportunity Mapping (Ulwick ODI)

### Outcome-Driven Innovation Process

Anthony Ulwick's Outcome-Driven Innovation (ODI) is a quantitative approach to
identifying innovation opportunities:

```
Step 1: Define the job-to-be-done
        "What job is the customer trying to accomplish?"

Step 2: Map the job steps (job map)
        Define --> Locate --> Prepare --> Confirm -->
        Execute --> Monitor --> Modify --> Conclude

Step 3: Identify desired outcomes for each job step
        "Minimize the time it takes to [outcome]"
        "Minimize the likelihood of [negative outcome]"

Step 4: Quantify opportunity (survey)
        Importance (1-10) and Satisfaction (1-10)

Step 5: Calculate opportunity score
        Opportunity = Importance + max(Importance - Satisfaction, 0)

Step 6: Prioritize: High importance, low satisfaction = biggest opportunity
```

### Opportunity Landscape

```
Satisfaction
    High |  OVER-SERVED        |  APPROPRIATELY SERVED |
         |  (table stakes,     |  (maintain,           |
         |   don't innovate    |   don't invest more)  |
         |   here)             |                       |
    -----+---------------------+-----------------------+
    Low  |  UNDERSERVED        |  UNIMPORTANT          |
         |  (INNOVATE HERE!)   |  (ignore --           |
         |  High importance,   |   low importance,     |
         |  low satisfaction   |   low satisfaction)   |
         |                     |                       |
         +---------------------+-----------------------+
              High                   Low
              Importance
```

## Additional Ideation Methods

### Worst Idea Technique
Generate deliberately terrible ideas, then invert them to find good ones.
This bypasses evaluation apprehension and often produces surprisingly creative results.

### Brainwriting (6-3-5 Method)
6 people write 3 ideas in 5 minutes, then pass to the next person who builds
on them. Eliminates production blocking. 108 ideas in 30 minutes.

### Assumption Storming
List every assumption about the problem/market/solution. Challenge each one.
Innovation often comes from violating an assumption everyone took for granted.

## Key Takeaways

1. **Structure beats spontaneity**: Systematic ideation outperforms unstructured brainstorming.
2. **Individual before group**: Silent individual generation, then group building.
3. **Quantity enables quality**: Generate 100 ideas to find 3 good ones.
4. **Cross-domain analogies drive breakthroughs**: Look outside your industry.
5. **10x thinking forces new approaches**: Incremental goals yield incremental ideas.
6. **JTBD reveals real opportunities**: Innovate where importance is high and satisfaction is low.
